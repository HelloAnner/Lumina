# Lumina UI 设计规范

## 一、设计理念

**关键词：极简、现代、克制、专注内容**

参考标杆：Claude Code 官网、Linear、Vercel Dashboard、Notion。

风格是那种「只有必要元素，每个像素都有意义」的现代 SaaS 感——不是传统企业软件的厚重感，也不是消费级 App 的花哨感。用户打开产品，第一感觉是"干净、快"，第二感觉是"这个工具懂我"。

核心原则：
- **内容优先**：UI 退到背景，文字和数据是主角
- **留白即设计**：大量呼吸空间，不堆砌元素
- **一致的节制**：色彩、动效都保持克制，不炫技
- **功能即美学**：每个 UI 元素都必须有实际用途

## 二、色彩系统

### 背景层次（深色模式为主，支持浅色）

| Token | 深色值 | 浅色值 | 用途 |
|-------|--------|--------|------|
| `bg-base` | `#0A0A0B` | `#FAFAFA` | 整体页面背景 |
| `bg-surface` | `#111113` | `#FFFFFF` | 卡片、面板背景 |
| `bg-elevated` | `#1A1A1E` | `#F4F4F5` | 悬浮元素、下拉菜单 |
| `bg-overlay` | `#222226` | `#E4E4E7` | hover 状态背景 |

### 文字层次

| Token | 深色值 | 浅色值 | 用途 |
|-------|--------|--------|------|
| `text-primary` | `#FAFAFA` | `#09090B` | 正文、标题 |
| `text-secondary` | `#A1A1AA` | `#71717A` | 次级说明、标签 |
| `text-muted` | `#52525B` | `#A1A1AA` | 占位符、禁用状态 |

### 强调色

| Token | 值 | 用途 |
|-------|----|------|
| `accent-primary` | `#8B5CF6` | 品牌色，主按钮、选中状态 |
| `accent-secondary` | `#6D28D9` | hover 深化 |
| `accent-subtle` | `#8B5CF620` | 轻量背景高亮 |

高亮颜色（书籍划线用）：
- 黄色：`#FBBF2440`（透明度高亮）
- 绿色：`#34D39940`
- 蓝色：`#60A5FA40`
- 粉色：`#F472B640`

### 边框

| Token | 深色值 | 浅色值 |
|-------|--------|--------|
| `border-default` | `#27272A` | `#E4E4E7` |
| `border-subtle` | `#18181B` | `#F4F4F5` |
| `border-focus` | `#8B5CF6` | `#8B5CF6` |

## 三、排版

**字体栈：** `Inter, -apple-system, BlinkMacSystemFont, sans-serif`

阅读器正文可切换为衬线：`Georgia, "Noto Serif SC", serif`

### 字号系统

| 用途 | 大小 | 字重 | 行高 |
|------|------|------|------|
| 页面大标题 | 24px | 600 | 1.3 |
| 章节标题 | 18px | 600 | 1.4 |
| 卡片标题 | 15px | 500 | 1.4 |
| 正文 | 14px | 400 | 1.6 |
| 说明/标签 | 12px | 400 | 1.5 |
| 微标注 | 11px | 400 | 1.4 |

---

## 四、间距系统

基础单位 4px，遵循 4/8/12/16/24/32/48/64 阶梯。

- **页面内边距：** 24px（内容区左右）
- **卡片内边距：** 16px
- **元素间距：** 8px（紧凑）/ 12px（默认）/ 24px（分组）
- **Sidebar 宽度：** 220px（固定）

## 五、组件规范

### Sidebar

```
宽度：220px，固定不可折叠（MVP）
背景：bg-surface
右侧：1px border-default 分隔线
顶部 Logo 区：48px 高
导航项：36px 高，12px 水平内边距
激活状态：bg-elevated + text-primary + 左侧 2px accent-primary 竖线
hover 状态：bg-overlay，过渡 150ms ease
图标：16px，Lucide Icons
```

### 卡片（书籍卡片等）

```
背景：bg-surface
边框：1px border-default
圆角：8px
hover：border 变为 border-focus（150ms），轻微上移 translateY(-2px)
阴影：默认无阴影（极简），hover 时加 0 4px 16px rgba(0,0,0,0.3)
```

### 按钮

```
Primary：bg accent-primary，白色文字，8px 圆角，高度 36px，水平 padding 16px
Secondary：bg-elevated，text-secondary，同圆角
Ghost：透明背景，hover bg-overlay
危险：#EF444420 背景，#F87171 文字
禁用：opacity 0.4
```

### 输入框

```
背景：bg-elevated
边框：1px border-default
聚焦：border-focus + 外发光 0 0 0 3px accent-subtle
圆角：6px
高度：36px
```

### 标签/Badge

```
小圆角 4px，12px 字号
阅读进度、状态等用此组件
```

## 六、布局结构

### 整体布局

```
┌──────────────────────────────────────────────────────┐
│  Sidebar (220px)  │  主内容区（剩余宽度，min 800px）  │
└──────────────────────────────────────────────────────┘
```

### 书库页

```
顶部：页面标题 + 右侧操作按钮（上传、筛选）
内容：书籍封面网格，4 列，16px gap
书籍卡片：封面（120×160px）+ 书名 + 作者 + 进度条
```

### 知识库页（三栏）

```
┌────────────┬──────────────┬──────────────────────┐
│ 观点树     │ 文章列表     │ 文章正文              │
│ 220px      │ 260px        │ 剩余                  │
│ 可折叠     │              │ 最大宽度 720px 居中   │
└────────────┴──────────────┴──────────────────────┘
```

文章正文最大宽度 720px，内容居中，保证阅读舒适度。

### 阅读器页（全屏覆盖）

```
顶部：极细 bar，左侧书名返回，右侧工具图标（排版、全屏）
左侧：页面/目录抽屉（可收起）
中央：渲染内容，白色/护眼/夜间背景
右侧：划线批注面板（可收起）
```

## 七、动效原则

- 过渡时长：**150ms**（hover/focus）、**200ms**（面板展开）、**300ms**（页面切换）
- 缓动函数：`ease`（默认）、`ease-out`（进入）、`ease-in`（离开）
- **禁止复杂动画**：无弹簧、无过度弹跳，只用 opacity 和 transform
- 图谱节点力导向：物理模拟关闭后保持静态，hover 才触发高亮动效

---

## 八、图标规范

统一使用 **Lucide Icons**，16px（行内）/ 20px（操作按钮）。
颜色跟随 text-secondary，hover 后变 text-primary。

---

## 九、空状态设计

每个空状态必须包含：
1. 一个简洁的图标（线框风格，64px）
2. 一行标题（"还没有书籍"）
3. 一行说明（"上传第一本 PDF 或 EPUB 开始阅读"）
4. 一个 Primary 按钮 CTA

背景不放插画，只用线框图标 + 文字，保持极简。

---

## 十、响应式策略

MVP 以 **≥ 1280px 桌面端**为主，不做移动端适配。
最小可用宽度 1024px，低于此宽度提示"建议使用桌面端"。
