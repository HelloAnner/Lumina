# 模块：笔记（notes）

---

## 1. 模块职责

跨全部来源（书籍 + 文章）的高亮和批注聚合视图，是 Lumina 的统一笔记出口。

用户在这里：
- 浏览、搜索自己的全部阅读笔记
- 查看 AI 生成的论点卡片
- 从笔记跳回原文
- 管理标签

## 3. 内容类型

笔记页展示两种内容，混合排列：

### 高亮条目

```
高亮原文
批注（如有）
颜色标识 · 来源（书名/文章标题 + 章节）· 时间 · [跳回原文]
```

### 论点卡片（NoteCard）

AI 生成或用户手动创建的结构化笔记，内容为 Markdown。

```
★ 标题
AI 生成 / 用户创建 · 时间
内容摘要（前 2 行）
关联高亮数量
```

点击展开卡片，查看完整内容，可编辑。

---

## 4. 搜索

### 关键词搜索

即时过滤：对高亮内容、批注文字、卡片标题/内容做全文匹配。

### 语义搜索

输入框右侧的「语义」切换：

```
invoke("search_highlights_semantic", { query, limit: 20 })
  → sqlite-vec cosine similarity
  → 返回按相关度排序的高亮列表
```

结果中显示相关度分数（可选）。

---

## 5. 过滤器

| 过滤维度 | 选项 |
|----------|------|
| 标签 | 全部 / 用户定义的标签 |
| 来源 | 全部 / 书籍 / 文章 / 某本具体书 |
| 颜色 | 全部 / 🟡 / 🟢 / 🔵 / 🩷 |
| 类型 | 全部 / 高亮 / 论点卡片 |
| 时间 | 全部 / 今天 / 本周 / 本月 |

---

## 6. 论点卡片管理

用户可手动新建卡片：

```
[+ 新建卡片]
  → 标题 + Markdown 内容编辑器
  → 可选关联已有高亮（搜索选取）
  → 添加标签
  → 保存
```

AI 生成的卡片（来自读后 AI）可编辑，编辑后标记为 `generated_by: user`，不再被 AI 覆盖。

---

## 7. 跳回原文

点击任意高亮条目的「跳回原文」：

```
invoke("open_reader", {
  sourceType: 'book' | 'article',
  sourceId,
  highlightId
})
```

阅读器打开后定位到该高亮位置并短暂闪烁。

---

## 8. Tauri 命令清单

| 命令 | 说明 |
|------|------|
| `get_highlights(filters?)` | 获取高亮列表（支持全部过滤维度） |
| `search_highlights_semantic(query, limit)` | 语义搜索高亮 |
| `get_note_cards(filters?)` | 获取论点卡片列表 |
| `create_note_card(data)` | 新建卡片 |
| `update_note_card(id, data)` | 更新卡片内容/标签 |
| `delete_note_card(id)` | 删除卡片 |
| `get_tags()` | 获取所有标签及使用数量 |
| `update_highlight_tags(id, tags)` | 更新高亮标签 |
