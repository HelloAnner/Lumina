# 阅读布局偏好

> 对应模块：阅读器 / 文章阅读器
> 状态：已实现

---

## 目标

让每个用户在每一本书、每一篇文章里都保留自己的阅读布局习惯，下一次进入时自动恢复，而不是每次都从默认展开状态开始。

当前需要记住两类偏好：

- 左侧目录是否收起
- 右侧笔记区域是否收起

其中：

- 书籍阅读器的左侧是目录（TOC）
- 文章阅读器的左侧是大纲
- 右侧统一视为笔记区域，承载高亮、批注与相关操作

## 状态模型

阅读布局偏好按「用户 × 资源」存储。

- `resourceType`：`book` | `article`
- `resourceId`：书籍 ID 或文章 ID

每条记录包含：

- `outlineCollapsed`：左侧目录 / 大纲是否收起
- `notesCollapsed`：右侧笔记区域是否收起

默认值：

- `outlineCollapsed = false`
- `notesCollapsed = false`

也就是首次进入保持完整三栏，之后按用户真实操作覆盖。

## 交互原则

- 收起和展开都要立即更新本地界面，不等待网络返回
- 后端保存使用轻量防抖，避免频繁点击带来多次写入
- 切换到另一篇文章或另一本书时，只读取该资源自己的偏好
- 不把 A 书的收起状态带到 B 书
- 共享只读页不写数据库，继续使用访客本地缓存

## 服务端约定

新增独立的阅读布局偏好存储，而不是塞进全局 UI 宽度表中。

原因：

- 宽度偏好是全局级别
- 收起状态是资源级别
- 两者生命周期和读取时机不同，拆开后更容易维护和扩展

接口语义：

- `GET /api/preferences/reader-layout`
- `PUT /api/preferences/reader-layout`

读取时由页面服务端入口直接拿初始值，客户端只负责后续更新。

## 恢复顺序

登录用户：

1. 读取数据库中的资源级偏好
2. 用该值初始化阅读器面板状态

共享只读用户：

1. 读取现有访客 localStorage
2. 不写数据库

这样可以保证“个人偏好”与“访客临时偏好”互不污染。
