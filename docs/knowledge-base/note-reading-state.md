# 笔记阅读状态与目录记忆

> 对应模块：知识库 / 笔记阅读区
> 状态：实现中

---

## 目标

让知识库里的每一篇笔记都拥有稳定、可恢复的阅读上下文：

- 点击笔记后，浏览器 URL 唯一指向当前笔记
- 刷新页面后，仍然停留在同一篇笔记
- 记住用户对该笔记目录的展开/收起习惯
- 记住该笔记的阅读位置，并在下次进入时恢复
- 阅读过程中，右侧目录始终悬浮，并高亮当前所在章节

## 状态模型

以 `noteKey` 作为唯一键，格式如下：

- `viewpoint:{id}`
- `imported:{id}`

每个用户、每个 `noteKey` 单独存储以下状态：

- `outlineCollapsed`：目录是否收起
- `scrollTop`：正文滚动位置
- `anchorHeadingId`：当前阅读锚点标题

## URL 约定

知识库页面统一使用查询参数表达当前选中项：

- 观点笔记：`/knowledge?viewpoint={id}`
- 导入笔记：`/knowledge?importedNote={id}`

同一时刻只保留一种选择，避免状态冲突。

## 交互原则

- 目录默认展开，但按“每个用户 × 每篇笔记”持久化
- 展开时，目录以右侧悬浮卡片形式存在
- 收起时，仅保留轻量入口按钮
- 目录高亮跟随正文滚动实时变化
- 切换笔记前，先保存当前阅读状态，再跳转到下一篇
