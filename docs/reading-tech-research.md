# 阅读体验技术调研

日期：2026-03-23

## 结论

当前 Web EPUB 阅读体验可以重点参考以下开源路线：

1. `epub.js`
   - 适合做底层渲染与定位
   - 支持 `paginated` 与 `scrolled` 两类流模式
   - 横向分页与纵向连续滚动都能落地

2. `react-reader`
   - 是 `epub.js` 的 React 包装
   - 现成支持 TOC、locationChanged、主题和 selection 扩展
   - 适合作为后续替换当前自研阅读壳的参考

3. `Paged.js`
   - 更适合做“版心排版 / 分页观感”增强
   - 如果后面要进一步强化书页感，可以把正文排版层与阅读引擎拆开考虑

4. `foliate-js`
   - 对电子书阅读体验更完整
   - 对连续滚动 / 分页切换的思路值得借鉴

## 当前项目采用策略

- `horizontal` 模式：保留单章分页阅读感
- `vertical` 模式：改为网页式连续滚动
- 目录、进度、划线、想法等业务状态继续由项目自己掌控

## 后续优化方向

- 把 EPUB 真实渲染升级到 `epub.js`
- 用 CFI / location 取代当前简化章节索引
- 提升目录树准确率与层级信息
- 强化正文排版：版心、段距、首行缩进、章节过渡、翻页动效
