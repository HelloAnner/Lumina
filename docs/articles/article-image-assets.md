# 文章图片资产

## 目标

让文章解析不仅保留正文，也保留正文中的图片，并尽量保持图片在正文中的原始位置。

同时，文章被永久删除或归档清理时，相关联的数据和图片资产一并删除，避免残留。

## 设计

### 图片位置保留

- 在进入 `Readability` 之前，对原始 HTML 做一次预处理
- 将正文中的图片节点替换为可被 `Readability` 保留的轻量占位标记
- `Readability` 输出后，再把这些占位标记还原为 `ArticleSection[type=image]`
- 这样正文与图片仍在同一条解析链上，图片位置尽量贴近原文

### 图片存储

- 解析出的文章图片不直接引用外链
- 服务端下载图片二进制并上传到 MinIO
- 文章内容中的图片 `src` 改写为 Lumina 自己的代理地址
- 分享页访问时，再改写为公开分享代理地址

### 清理范围

文章永久删除或归档过期清理时，一并删除：

- 文章本身
- 文章高亮
- 文章翻译缓存
- 文章分享链接
- 文章图片资产（MinIO）

## 接口

- `GET /api/articles/:id/assets/:assetId`
- `GET /api/shares/public/:token/articles/assets/:assetId`

## 约束

- 仅下载 `http/https` 图片
- 非图片内容类型直接跳过
- 下载失败不阻塞文章入库，保留原始图片 URL 兜底
