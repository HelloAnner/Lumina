# 来源文件夹视图

## 目标

把文章库首页改成“一个来源一个文件夹”的入口层。用户先按来源进入，再在来源内部集中浏览和清理文章。

## 信息架构

- 文章首页只展示来源文件夹，不直接混排所有文章
- 一个 Scout 信息源对应一个固定文件夹
- 文件夹本身不可删除；来源的新增、暂停、删除仍在信息源页完成
- 进入文件夹后，展示该来源下的全部文章
- 手动添加的文章使用系统文件夹 `手动添加` 承载，避免游离在来源体系之外

## 交互

- 首页卡片展示：
  - 来源名称
  - 来源地址或简述
  - 当前文章数
  - 最近抓取时间
- 点击卡片进入 `/articles?sourceId={id}` 对应的来源视图
- 来源内列表支持多选
- 提供 `全选`、`清除选择`、`删除所选文章`
- 删除文章是文章级操作，不影响来源文件夹和来源配置

## 数据规则

- 来源文件夹由 `ScoutSource` 和 `ScoutArticle.sourceId` 联合生成
- 若文章来自不存在于 `ScoutSource` 表中的系统来源（如 `manual`），仍需生成只读文件夹
- 来源文件夹统计默认只计算未归档文章
- 删除所选文章走批量永久删除，并同步清理图片等派生资源

## 视觉原则

- 首页是安静的文件夹网格，不出现来源删除按钮
- 来源内列表采用低干扰表格视图，强调勾选与批量操作
- 危险操作集中在顶部批量工具条，不在每个来源卡片上制造紧张感
